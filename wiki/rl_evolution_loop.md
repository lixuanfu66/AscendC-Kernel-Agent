# 强化学习与进化闭环

## 目标

把 CANN 算子编程过程建模为可评估、可反馈、可积累经验的序列决策问题，让 Agent 在解决任务的同时产生更好的经验数据，并利用这些数据改进后续任务表现。

### 奖励

| 信号 | 奖励方向 |
| --- | --- |
| 编译通过 | 正 |
| 正确性通过 | 强正 |
| 性能提升 | 强正，按相对提升或达标程度计算 |
| 编译/运行次数过多 | 负 |
| 重复同类失败 | 负 |
| 产生可复用经验 | 正 |
| 错误泛化或无依据结论 | 负 |

## 训练路线

### 路线 1：离线轨迹学习

收集真实开发轨迹，清洗为状态、动作、观察、奖励序列，用于 SFT、行为克隆或过程奖励模型训练。

优点：实现难度较低，能先利用现有实验数据。

风险：轨迹质量决定上限，需要避免把低质量 trial-and-error 学进去。

### 路线 2：偏好优化

对同一任务的多条轨迹进行比较，标注哪条更好。偏好可来自：

- 更少迭代。
- 更高性能。
- 更清晰诊断。
- 更可复用经验。

### 路线 3：自动任务生成与 self-play

从已有算子和规格变体生成任务，让 Agent 自动尝试、评测、总结经验。

关键问题：

- 任务必须可验证。
- 不能只优化少数固定 shape。
- 自动生成的经验要经过质量过滤。

## 经验库参与闭环

1. 新任务开始时检索相关经验。
2. Agent 使用经验形成计划或假设。
3. 任务结束后比较经验是否有效。
4. 更新经验置信度。
5. 高价值经验进入训练数据，低价值经验降权或归档。
