# 昇腾 CANN 算子编程 Agent 研究计划

## 目标声明

本项目研究面向昇腾 CANN 算子编程的 Agent 系统，目标是形成一个能够利用知识库、遵循算子开发流程、自动编译执行与性能分析、沉淀优化经验，并通过强化学习或数据闭环持续进化的研究与工程框架。

核心问题：

1. 知识库：哪些知识对 CANN 算子编程 Agent 有帮助，如何组织、检索、更新与治理。
2. 编程流程：如何覆盖算子规格、计算流程分析、planning、coding、编译、执行、性能测试、问题分析、修复与优化的多轮迭代。
3. 经验库：如何把检索、推理、调试、优化过程中的有效轨迹沉淀为可复用经验。
4. 强化学习闭环：如何利用知识库与经验库，让 Agent 产生更多高质量经验，并反过来提升算子实现能力。
5. 训练语料：如何把经验、推理轨迹和任务过程整理为 Agent 基础模型增量训练语料。

状态：in_progress，已有初版框架

任务：

- 明确 CANN 算子编程 Agent 的输入、输出、约束和评价指标。
- 梳理知识库类型：官方文档、API/DSL 参考、样例代码、编译错误案例、性能优化指南、硬件架构知识、历史轨迹。
- 设计知识组织方式：文档分层、元数据、任务类型标签、算子类型标签、错误类型标签、优化策略标签。
- 设计个人 wiki/知识库目录结构，兼容长期研究笔记和 Agent 可检索资料。

产物：

### 阶段 2：算子编程 Agent 工作流设计

状态：pending

任务：

- 定义从算子规格到可运行实现的标准流程。
- 拆分 Agent 子能力：规格解析、计算图/tiling 分析、代码生成、编译诊断、运行验证、性能 profiling、优化建议、迭代控制。
- 定义每个阶段需要的知识检索输入、检索输出和决策记录格式。
- 设计失败处理协议：编译失败、运行错误、精度错误、性能退化、资源约束失败。

产物：

### 阶段 3：经验库与推理轨迹沉淀机制

状态：pending

任务：

- 定义“经验”的最小结构：场景、症状、假设、检索依据、尝试、结果、结论、适用边界。
- 区分经验类型：编译修复、性能优化、API 使用、tiling 策略、数据搬运、并行划分、精度验证。
- 设计经验入库标准：成功经验、失败经验、反例、可泛化程度、置信度。
- 设计经验检索策略：按错误、算子类型、性能瓶颈、硬件资源、代码模式检索。

产物：

- 经验库 schema。
- 经验条目模板。
- 推理轨迹清洗和压缩规则。

### 阶段 4：强化学习/进化闭环方案

状态：pending

任务：

- 定义奖励：正确性、编译通过、性能提升、迭代成本、经验可复用性。
- 研究离线 RL、偏好优化、过程奖励模型、self-play/自动任务生成等可行路线。

产物：
